## 🎯 快速部署步骤

### 第一步：在GitHub创建新仓库

1. 登录你的GitHub账号
2. 点击右上角 "+" → "New repository"
3. 仓库名称填写：`fund-tracker`（或你喜欢的名字）
4. 选择 "Public"（这样才能免费使用GitHub Pages）
5. 勾选 "Add a README file"
6. 点击 "Create repository"

### 第二步：上传文件到GitHub

创建仓库后，你需要将以下文件上传到仓库根目录：

**方法A：使用GitHub网页上传（推荐新手）**

1. 进入你的新仓库
2. 点击 "Add file" → "Upload files"
3. 将以下文件拖拽上传：
   - `fetch_fund_data.py`
   - `index.html`
   - `requirements.txt`
   - `README.md`
4. 创建文件夹 `.github/workflows/`，然后上传 `update.yml` 到这个文件夹

**方法B：使用Git命令行（推荐）**

```bash
# 在你的本地电脑上（需先安装Git）
# 1. 克隆仓库
git clone https://github.com/你的用户名/fund-tracker.git
cd fund-tracker

# 2. 将我创建的文件复制到这个目录

# 3. 提交并推送
git add .
git commit -m "初始化基金追踪系统"
git push origin main
```

### 第三步：启用GitHub Pages

1. 进入你的仓库
2. 点击 "Settings"（设置）
3. 在左侧菜单找到 "Pages"
4. Source 选择 "GitHub Actions"
5. 保存

### 第四步：启用GitHub Actions

1. 进入仓库的 "Actions" 标签
2. 你会看到 "每日更新基金数据" 工作流
3. 点击 "Enable workflow" 启用它

### 第五步：手动运行一次测试

1. 在 "Actions" 标签中
2. 选择 "每日更新基金数据" 工作流
3. 点击 "Run workflow" → "Run workflow"
4. 等待运行完成（约1-2分钟）
5. 检查 `data/funds_data.json` 文件是否生成

### 第六步：访问你的基金页面

部署成功后，访问：
```
https://你的用户名.github.io/fund-tracker/
```

## ⚙️ 自定义设置

### 修改更新时间

编辑 `.github/workflows/update.yml`，找到这一行：

```yaml
- cron: '0 10 * * *'
```

这是UTC时间，北京时间需要+8小时。例如：
- 北京时间每天18:00 → UTC 10:00（已设置）
- 北京时间每天20:00 → UTC 12:00，改为 `'0 12 * * *'`

### 添加/删除基金

## 🔍 常见问题

### Q1: 页面显示"数据加载失败"怎么办？

**原因**：`data/funds_data.json` 文件不存在或未生成。

**解决**：
1. 检查GitHub Actions是否运行成功
2. 进入 "Actions" 标签查看运行日志
3. 手动触发一次运行

### Q2: 我想在本地测试脚本怎么办？

**步骤**：

1. 安装Python（如果还没有）
2. 安装依赖：`pip install requests`
3. 运行脚本：`python fetch_fund_data.py`
4. 检查 `data/` 文件夹是否生成 `funds_data.json`
5. 用浏览器打开 `index.html` 查看效果

### Q3: 如何强制重新计算累计收益？

当前脚本使用历史净值自动计算。如需更精确的收益计算（考虑分红、拆分等），需要：
1. 记录你的实际购买金额和份额
2. 修改脚本加入个人持仓数据
